범인은 아니 문제에 대한 설명은 에러 메시지에 있을 확률이 가장 높으니 위 systemctl status kubelet의 결과를 찬찬히 살펴보니 swap에 대한 내용이 눈에 띈다.

그래서 바로 해결 시도.

$ sudo swapoff -a
